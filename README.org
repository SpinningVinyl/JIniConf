* JIniConf

JIniConf is a small Java 17 library for reading, modifying, and writing =.ini= and =.conf= files.

All property values are stored as Java strings. Nested sections are represented by nested ~IniConf~ objects.

** Supported format

Property keys and each component of a section path must contain one or more ASCII word characters: letters, digits,
or underscores. Section path components are separated by dots.

#+begin_src ini
root_key = value

[server]
host_name = localhost

[server.http]
port = 8080
#+end_src

Keys and section names are converted to lowercase using ~Locale.ROOT~ before storage. Query arguments are normalized
in the same way, so key and section lookup is case-insensitive.

#+begin_src java
iniConf.put("Server.HTTP", "Host_Name", "localhost");
String host = iniConf.get("SERVER.HTTP", "HOST_NAME");
#+end_src

Lines whose first non-whitespace character is ~;~ or ~#~ are comments. Blank lines are ignored. Every other line must
be a valid section header or property; malformed input causes an ~IniConfFormatException~ containing the input line
number. Invalid values supplied through the programmatic API, such as values containing NUL, cause an
~IllegalArgumentException~.

*** Value rules

Values must not be ~null~ and must not contain NUL or line-terminator characters. Leading and trailing whitespace is
removed with ~String.strip()~ before a value is stored. Internal whitespace is preserved. A value containing only
whitespace therefore becomes an empty string.

The serializer uses the following canonical representation:

- Simple values are written without quotes.
- Empty strings are written as ~""~.
- Values containing whitespace, a double quote, or a backslash are enclosed in double quotes.
- Within a quoted value, ~"~ is encoded as ~\"~ and ~\~ is encoded as ~\\~.
- Single quotes are ordinary value characters.

#+begin_src ini
simple = value
empty = ""
message = "He said \"hello\""
path = "C:\\config\\application.ini"
#+end_src

When parsing quoted values, only ~\"~ and ~\\~ are accepted escape sequences. Unknown or incomplete escape sequences,
unescaped double quotes, and unquoted backslashes are rejected.

** Reading and parsing files

~IniConfReader~ reads files as UTF-8. A leading UTF-8 byte-order mark is removed before parsing.

#+begin_src java
Optional<IniConf> result = IniConfReader.read("sample.ini");
if (result.isPresent()) {
    IniConf iniConf = result.get();
    // use the configuration
}
#+end_src

~IniConfReader.read()~ returns ~Optional.empty()~ when an I/O error occurs, the file is missing, or the input is not
valid UTF-8. Invalid INI content causes an ~IniConfFormatException~.

An already-loaded Java string can be parsed directly:

#+begin_src java
IniConf iniConf = new IniConf(input);
#+end_src

The direct constructor operates on Unicode Java character data and therefore performs no byte decoding or BOM
handling.

** Querying configurations

Use ~isEmpty()~ to check whether a configuration contains any properties or sections.

Use ~isKey()~, ~get()~, and ~getOrDefault()~ for root properties:

#+begin_src java
if (iniConf.isKey("root_key")) {
    String value = iniConf.get("root_key");
}

String value = iniConf.getOrDefault("missing_key", "default_value");
#+end_src

The overloads accepting a section path query nested properties:

#+begin_src java
if (iniConf.isKey("server.http", "port")) {
    String port = iniConf.get("server.http", "port");
}

String port = iniConf.getOrDefault("server.http", "port", "8080");
#+end_src

Use ~isSection()~ and ~getSection()~ to query whole sections:

#+begin_src java
IniConf httpSection = null;
if (iniConf.isSection("server.http")) {
    httpSection = iniConf.getSection("server.http");
}
#+end_src

~getProperties()~ and ~getSections()~ return unmodifiable map views:

#+begin_src java
Map<String, String> properties = iniConf.getProperties();
Map<String, IniConf> sections = iniConf.getSections();
#+end_src

** Creating and modifying configurations

Create an empty configuration:

#+begin_src java
IniConf iniConf = new IniConf();
#+end_src

Add root and nested properties with ~put()~:

#+begin_src java
iniConf.put("key", "value");
iniConf.put("section.subsection", "key", "value");
#+end_src

The nested overload creates missing sections before inserting the property. Section creation and property insertion
are separate operations: if the key or value is invalid, sections created while resolving the path remain.

Add or replace a section by supplying its ~IniConf~ object:

#+begin_src java
iniConf.addSection("section1", new IniConf());
iniConf.addSection("level1.level2.level3", new IniConf());
#+end_src

Section objects cannot be ~null~, reused within the same configuration graph, or added in a way that creates a cycle
or repeated section identity.

** Serializing and writing files

~toString()~ returns the INI representation of a configuration:

#+begin_src java
String iniString = iniConf.toString();
#+end_src

The relative order of properties and sections is unspecified. Parsing serialized output reconstructs the same
configuration within the supported identifier and value contract.

~IniConfWriter~ writes the serialized representation as UTF-8 without a byte-order mark:

#+begin_src java
boolean written = IniConfWriter.write("sample.ini", iniConf);
if (!written) {
    // handle the I/O failure
}
#+end_src

The method returns ~false~ if an I/O error occurs and throws ~NullPointerException~ if the filename or configuration
is ~null~.
